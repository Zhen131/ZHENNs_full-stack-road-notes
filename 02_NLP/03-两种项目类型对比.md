## 两条路是什么 · The Two Paths

### 🅰️ Type 1 · Build an LLM-based **Multi-Agent**
> 用至少两个本地 (`Ollama`) agent,把一个复杂问题拆解协作完成,并证明它**打赢单 prompt 的 baseline**。

**本质**:你不训练模型,而是**编排 (orchestrate)** 现成的本地 LLM。比拼的是 **agent 设计 + 工作流 + prompt 工程 + 对比实验**。

### 🅱️ Type 2 · Create (Train/Fine-tune) an **efficient LLM**
> 自己搞数据,**训练或微调 (train / fine-tune)** 一个高效的 LLM,评估后打包成应用。

**本质**:这是一条**机器学习工程 (ML engineering)** 路线。比拼的是 **数据处理 + 训练 + benchmark + 工程化**。

---

## 📊 正面对比表 · Head-to-Head

| 维度 | 🅰️ Type 1 (Multi-Agent) | 🅱️ Type 2 (Train/Fine-tune) |
| --- | --- | --- |
| **你实际在构建什么** | agent 之间的 `orchestration` 工作流 | 一个训练/微调出来的 LLM + pipeline |
| **核心技能** | prompt 设计、agent 角色拆分、`Ollama` 调用、对比实验设计 | 数据收集清洗、训练/微调、`benchmark`、打包 |
| **算力 / 硬件** | 低 —— 本地跑 inference 即可,普通电脑能扛小模型 | **高** —— 训练/微调通常吃 GPU,`at least 3 models` 更费算力 |
| **数据需求** | 几乎不用准备数据集 | **必须**有自己的数据 + `EDA` + 预处理 |
| **拿 5 分的关键动作** | 原创 idea + `iterative feedback loop` + 参数理论 + **significantly** 打赢 baseline | **≥3 个模型** + `complex benchmarking` + 有深度的 `conclusion` |
| **最大风险 ⚠️** | 「怎么**量化证明** multi-agent 真的更好」很难做扎实 | 算力/训练踩坑多,3 个模型工作量大,容易做不完 |
| **和你的方向契合度** | 高 —— 直接练 **AI Agent 开发** | 中 —— 练 ML / 训练功底 |
| **能复用到别处** | agent 编排经验、prompt 工程 | 训练/微调、数据工程经验 |

## 🧗 各自「拿 5 分」的真正难点

### Type 1 的难点:证明「显著更好」
rubric 里 `clear proof that the multi-agent approach significantly outperformed the baseline` 这句话是关键。做出两个 agent 不难,**难在拿出有说服力的对比数据**:

- 要设计一个**能量化**的任务(比如代码任务看 `code quality`、问答任务看 `accuracy`、事实任务看 `hallucination reduction`)。
- 要让 baseline 和 multi-agent **在同样输入下**跑,公平对比。
- 要做出 **iterative feedback loop**(agent 互相改),而不是两个 agent 各跑一次拼起来。

### Type 2 的难点:算力 + 3 个模型 + 跑完整 pipeline
- `Use more models, at least 3` —— 5 分硬要求,意味着**训/调 3 个模型**并对比,工作量直接 ×3。
- 训练/微调对**硬件和调参经验**要求高,容易卡在环境和 GPU 上。
- 要把 **EDA → 预处理+UnitTest → 训练 → complex benchmark → 打包应用 → conclusion** 整条链都做扎实。
- 注意题目要 **efficient**(高效)—— 暗示走 `Efficient Adaptation`(如 LoRA 一类轻量微调,见 [[04-课程内容地图]] 第 VII 讲),而不是从零训练大模型。

## 🤔 帮你自问的几个问题 · Decide for Yourself

不替你选,但回答这几个问题,答案会很清楚:

1. **你手上有 GPU / 够用的算力吗?** 没有 → Type 1 友好得多。
2. **你更想练「agent 编排」还是「模型训练」?** 你的选课方向写的是 **AI Agent 开发** —— 但这只是契合度,不是强制。
3. **你能不能找到/造出一份合适的数据集?** 造不出来 → Type 2 第一步就卡住。
4. **你独立做还是组队?** 独立做又想拿 5 分,Type 1 的「单 prompt vs multi-agent」对比相对更可控;Type 2 的「3 个模型 + 完整 pipeline」单人工作量很重。
5. **哪条路的「5 分门槛」你更有把握跨过去?** 回去对照 [[02-评分标准与拿5分策略]] 的 `Grade 5` 那一行再判断。

> [!tip] 一个中立观察
> 两条路都能拿 5 分。Type 1 的**入门门槛和算力门槛更低、和你的 Agent 方向更顺**,但「证明显著更好」需要动脑设计实验;Type 2 **技术含金量高、ML 功底涨得快**,但算力和工作量是硬成本。先确认算力和数据这两个「硬约束」,再谈兴趣。
