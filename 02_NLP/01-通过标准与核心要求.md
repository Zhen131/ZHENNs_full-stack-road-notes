## 🚦 交付物总清单 · Deliverables Checklist

把这个清单当成「提交前最后检查表」:

- [ ] **出勤达标** —— `Fewer absences than allowed` + `Active participation`(缺勤少于上限 + 课堂积极参与)
- [ ] **一个能跑的 application** —— 解决一个**真实问题 (real problem)**,用课上学的 solutions / models
- [ ] **一段 video**,长度 **5-10 分钟**,大小 **≤ 50 MB**
- [ ] video 里**每个成员讲自己的贡献** (3-8 分钟)
- [ ] video **结尾演示 application 实际运行** (1-2 分钟)
- [ ] **一份干净的 `.ipynb` notebook** —— `clear (without executed blocks)`,包含全部 Python 代码
- [ ] **所有 agent 用 Ollama 本地运行** (local execution)
- [ ] **至少两个 agent**,各有独立 `system instruction`
- [ ] **和 baseline 做对比** (multi-agent vs 单 prompt)

## 📹 Video 规格 · 逐条拆解

考试用语**原文保留**,旁边是中文解读:

| 英文要求 (原文) | 中文解读 | 易错点 ⚠️ |
| --- | --- | --- |
| `Maximum length of video is 5-10 minutes` | 视频 5-10 分钟 | 别超时,也别太短 |
| `Each creator must present their own contribution (for 3-8 minutes)` | **每个人**讲自己做的部分 | 单人做也要自己出镜讲清贡献 |
| `The application must be shown in action at the end (for 1-2 minutes)` | 结尾留 1-2 分钟**实际跑一遍** | 必须有「real demo」,不能只讲 PPT |
| `Video size must be 50 MB or less` | 文件 ≤ 50 MB | 录屏要压缩;50MB 装 10 分钟视频需要压制 |

> [!warning] 50 MB 是个很紧的限制
> 10 分钟视频压到 50 MB 以下,意味着分辨率/码率要控制。建议:**1080p → 720p**、用 `H.264` 编码、关掉无谓的高帧率。录完先查文件大小再说。

## 📓 Notebook 规格 · 关键细节

> [!important] `clear (without executed blocks)`
> 提交的 `.ipynb` 必须是**没有执行过 output 的干净版本**。也就是说:写完调试好之后,**Kernel → Restart & Clear Output**,再保存提交。带着一堆运行结果的脏 notebook 不符合要求。

- 一份 notebook 包含**全部项目代码** (`all the project code`)
- 语言:Python
- 状态:clear,无已执行 cell 输出

## 🤖 三个「Type 1 味道」的核心要求

下面三条在原始要求里和上面并列,但它们描述的其实是 [[03-两种项目类型对比|Project Type 1 (multi-agent)]] 的框架。如果你走 [[03-两种项目类型对比|Type 2 (train/fine-tune)]],评分看的是另一张 task 表(见 [[02-评分标准与拿5分策略]])。

### 1. Local Execution(本地运行)
> `All agents must run locally using Ollama.`

所有 agent **必须用 Ollama 在本地跑**。不许用云端 API(OpenAI / Claude API 之类)。这是会直接导致 `Grade 1 (Fail)` 的红线 —— rubric 原文:`The system does not run locally ... → Fail`。

### 2. Multi-Agent Design(多智能体设计)
> `Use at least two agents with unique system instructions (e.g., a Researcher and a Technical Writer, or a Coder and a Reviewer).`

- **至少两个 agent**
- 每个 agent 有**独立的 `system instruction`**(角色不同、指令不同)
- 例子:`Researcher` + `Technical Writer`,或 `Coder` + `Reviewer`

### 3. Comparative Analysis(对比分析)
> `You must compare the multi-agent output against a "Baseline" (a single prompt sent to a standard chat agent).`

- 必须把 multi-agent 的结果和 **baseline** 比
- **baseline = 一个单 prompt 发给标准 chat agent** 的结果
- 这是证明「[[00-NLP课程总览|agentic workflow]] 比单 prompt 强」的关键动作

## 👥 团队与给分规则 · Team Rules

- **组队规模**:1-4 人 —— `Organizing into teams (2-4 people) or working individually`(可单独做)
- **非均分**:`It is not certain that the team members receive a uniform grade, but they get grades proportionate to the task they have completed`
  - 中文:同组**不一定拿一样的分**,按各自完成的任务量给分。
  - 含义 ⚠️:如果组队,**你自己那部分必须分量足够 + 在 video 里讲清楚**,否则贡献不够会被压分。
