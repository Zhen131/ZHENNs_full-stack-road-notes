## 🅰️ Project Type 1 —— Build an LLM-based multi-agent (自己的 idea)

逐档**英文原文 + 中文解读**。注意高分档是**层层叠加**的:5 分 = 4 分的一切 + 更强的东西。

### 🥇 Grade 5 (Excellent) —— 我们的目标
> `Outstanding original idea. Robust orchestration (e.g., iterative feedback loops between agents). Deep theoretical understanding of parameters (temperature, context window) and clear proof that the multi-agent approach significantly outperformed the baseline.`

拆成 4 个**必须同时做到**的点:

1. **`Outstanding original idea`** —— 出彩的**原创** idea,不是烂大街的「Writer + Editor 写文章」。
2. **`Robust orchestration` + `iterative feedback loops between agents`** —— 强健的**编排**,关键词是 **iterative feedback loop**(agent 之间反复来回、互相修正,而不是一条直线走完)。
3. **`Deep theoretical understanding of parameters (temperature, context window)`** —— 对 **`temperature`**(随机性/创造性)、**`context window`**(上下文窗口)这些参数有**深入理解**,并能讲出为什么这么设。
4. **`clear proof that the multi-agent approach significantly outperformed the baseline`** —— **清晰证明** multi-agent **显著优于** baseline(`significantly` 这个词很重要,要有说服力的证据)。

### 🥈 Grade 4 (Good)
> `Complex interaction logic. Agents provide critical feedback to one another. Detailed comparison with the baseline using specific metrics (e.g., accuracy, code quality, or hallucination reduction).`

- 复杂的交互逻辑 `complex interaction logic`
- agent 之间能**互相给批判性反馈** `critical feedback`
- 用**具体 metrics** 做详细对比:`accuracy`(准确率)、`code quality`(代码质量)、`hallucination reduction`(幻觉减少)

### 🥉 Grade 3 (Fair)
> `Clearly defined roles with distinct System Prompts. The notebook is well-documented and the comparison between the "single chat" and "multi-agent" results is present.`

- 角色清晰,各有不同的 **`System Prompt`**
- notebook **文档写得好** (well-documented)
- 「single chat」vs「multi-agent」的对比**存在**(有就行,不要求深)

### Grade 2 (Pass)
> `Basic implementation of two agents (e.g., Writer + Editor). The code runs on Ollama, but agent differentiation is minimal. The video explains the "how" but struggles with the "why" regarding agentic workflows.`

- 两个 agent 的基础实现(如 `Writer + Editor`)
- 能在 Ollama 跑,但 **agent 区分度很低** `agent differentiation is minimal`
- video 讲清了「**how 怎么做**」,但讲不清「**why 为什么用 agentic workflow**」⚠️

### Grade 1 (Fail) —— 红线,千万别踩
> `The system does not run locally, uses only one agent, or lacks the required video/notebook components. The submission was after the deadline.`

四个**任何一个**都直接挂:
- 系统**没在本地跑** (没用 Ollama)
- **只用了一个 agent**
- 缺 video / notebook 组件
- **过了 deadline 提交**

> [!danger] Type 1 拿 5 分的「分水岭」
> 2→3 的坎是 **agent 区分度 + 文档**;3→4 的坎是 **具体 metrics 对比 + 互相反馈**;4→5 的坎是 **原创 idea + iterative feedback loop + 参数理论 + 「significantly 显著」打赢 baseline**。盯死「4→5」那一行,那才是真正决定 Excellent 的地方。

## 🅱️ Project Type 2 —— Create (Train/Fine-tune) an efficient LLM

这套是**任务表**,5 个 task 各有 `Grade 3 / 4 / 5` 的要求。**读法**:高分**累加** —— 想拿 5 分,要先做到所有 task 的 Grade 3,再叠加 Grade 4 项,再叠加 Grade 5 项。

| Task | `Grade 3`(基础) | `Grade 4`(加强) | `Grade 5`(冲刺) |
| --- | --- | --- | --- |
| **Data** | `You need data [for] your project` 准备好项目数据 | `Exploratory data analysis (Visualization)` 做 **EDA** + 可视化 | (随 4 分一并满足) |
| **Preprocess** | `The code should include preprocessing steps (Curate the data)` 代码含**预处理**、清洗数据 | `UnitTest, Visualization` 加**单元测试** + 可视化 | (随 4 分一并满足) |
| **Model** | `Train a LLM model on the data` 在数据上**训练**一个 LLM | —— | `Use more models, at least 3` **至少 3 个模型** |
| **Evaluate** | `Evaluate your model on your test data` 在 test data 上**评估** | `Complex Benchmarking` **复杂 benchmark** | (随 4 分一并满足) |
| **Application** | `Package your model into an application. Present your pipeline and use the technical term / keywords correctly` 打包成**应用** + 正确使用术语 | —— | `Create a conclusion. Explain how your model works. Present the limitations and opportunities for further development` 写**结论** + 讲清模型原理 + 谈**局限与未来方向** |

> [!important] Type 2 拿 5 分 = 做满这三条
> 1. **Model**:至少用 **3 个模型** (`at least 3`) 对比。
> 2. **Application**:有 **conclusion**,解释模型怎么工作,讲 **limitations & opportunities**。
> 3. 同时把所有 task 的 Grade 3 基础 + Grade 4 加强(`EDA`、`UnitTest`、`Complex Benchmarking`)全部补齐。

> [!note] 一个需要跟老师确认的点
> Type 2 表里 `Grade 5` 只在 **Model** 和 **Application** 两行写了明确要求,其余几行空着。我的理解是:**空格 = 沿用 Grade 4 的要求**(高分累加)。开课后建议跟老师确认这个累加读法,以免理解偏差。

## 🎯 两条路的「拿 5 分」一句话总结

- **走 Type 1**:原创 idea + **iterative feedback loop** 编排 + 吃透 `temperature`/`context window` + 拿出 **significantly 打赢 baseline** 的硬证据。
- **走 Type 2**:**≥3 个模型** + 完整 pipeline(EDA → 预处理+UnitTest → 训练 → complex benchmark → 打包应用)+ 有深度的 **conclusion**(原理 + 局限 + 展望)。
