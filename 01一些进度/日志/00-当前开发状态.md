## 已完成

- 建立 `models / calculators / validators / utils / services / repositories / adapters` 分层骨架。
- 使用 `DecimalString → decimal.js → 展示格式化` 处理数量和金额精度。
- 实现 `decimalMath` 统一运算入口。
- 实现 `positionCalculator`：
  - 按时间排序和按资产聚合。
  - 买入、部分卖出、剩余成本和加权平均成本。
  - `realizedPnl`。
  - 同资产、同币种按 `recordedAt` 选择最新 `PriceSnapshot`。
  - `recordedAt` 相同时使用数组中后出现的快照。
  - 推导 `latestPrice`、`marketValue` 和 `unrealizedPnl`。
  - 无匹配快照时三个可选字段保持 `undefined`。
  - 零仓位卖出和超卖防御。
- 实现 `tradeValidator`：
  - 类型、资产、数量、价格、总额和手续费校验。
  - `quantity × price` 与 `totalValue` 容差校验。
  - 无持仓卖出、超卖和等量清仓校验。
- 将核心测试统一到 Vitest。

- 页面现有资产和交易是占位数据，不能视为真实账本状态。
- `PriceSnapshot` 当前只作为结构化输入参与核心测试；价格表单、价格校验和内存状态接入属于 Week 3。
