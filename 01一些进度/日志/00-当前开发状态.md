- 建立源码仓库根目录第一轮判断：
  - `README.md`、`package.json`、`src/` 是优先入口。
  - `.git/`、`.next/`、`node_modules/` 基本不作为学习入口。
  - 各类 config 文件先知道用途，遇到对应问题再深入。
- 理解 `node_modules/`、`package.json`、`package-lock.json`、`npm`、`dependencies`、`devDependencies` 的基本关系：
  - `package.json` 是项目身份、命令菜单和依赖清单。
  - `node_modules/` 是 npm 安装到本地的依赖实体。
  - `package-lock.json` 是 npm 维护的精确依赖版本锁，不人工细读。
- 初步建立 `Node.js / npm / package` 心智模型：
  - `Node.js` 是让 JavaScript 在浏览器外运行的环境。
  - `npm` 是 Node 生态的包管理器。
  - `package` 是可安装的代码工具包。
- 初步建立 `React / Next.js / TSX / CSS / Tailwind` 心智模型：
  - `React` 用于组织 UI component。
  - `TSX` 是 TypeScript + JSX 的组件/页面写法。
  - `Next.js` 是基于 React 的应用框架，规定页面、布局、路由、启动和构建规则。
  - `.next/` 是 Next.js 运行或构建后的生成物，不是源码。
  - `Tailwind CSS` 是通过 className 写样式的 CSS 工具体系。
