## 5. 跨周 Gate

### Gate A：React 内存态

通过周：Week 6。

红灯后果：Week 7 禁止开始 IndexedDB。

### Gate B：IndexedDB

必须满足：

- 明确整账 load/replace/clear 契约。
- hydration 完成前禁止持久化。
- 连续异步写不乱序。
- 写失败保留旧数据。
- 页面、service、reducer 不直接操作 IndexedDB。

红灯后果：Week 8 禁止开始导入导出。

### Gate C：备份恢复

必须满足：

- 版本化 backup envelope。
- 校验 `schemaVersion / assets / trades / priceSnapshots / feeRules`。
- 导入使用原子替换。
- 坏文件、未知版本或写入失败时原账本不变。

红灯后果：Week 9 禁止切换真加密。

### Gate D：加密与离线

必须满足：

- PBKDF2 + AES-256-GCM。
- salt、随机 IV 和 KDF 参数保存正确。
- Week 7 明文数据有首次迁移或明确清空/恢复策略。
- 正确密码可解锁，错密码和篡改失败。
- 会话密钥不写入持久化存储。
- 硬离线重启可用。

红灯后果：优先修复，不用图表或 benchmark 掩盖安全问题。

### Gate E：图表

必须满足：

- 两张保留图表均来自真实 `LedgerData` 派生数据。
- 历史曲线先定义横纵轴、估值时点、缺失价格和禁止未来信息规则。
- 图表数字与资产汇总一致。

### Gate F：benchmark pilot

必须满足：

- 固定 seed。
- 固定 100 / 1k / 10k 基础量级。
- 明确单笔写、批量预置、warm-up、清库和计时边界。
- 测量写入、查询、全量 P&L 重算和冷启动。
- 一条命令输出原始数据、环境信息和中间失败。

红灯后果：Week 13 final 禁止启动。

### Gate G：release candidate

通过周：Week 12。

必须满足：

- P0 回归全绿。
- test / lint / build 有新鲜验证证据。
- 导入回滚、解锁/篡改、刷新持久化和硬离线通过。
- 可复现 README 能从零启动应用并运行 benchmark。

### Gate H：最终 benchmark 与冻结

- 先固定 release candidate，再跑最终数据。
- final 开始后不修改行为代码。
- 若必须修改，受影响实验、统计图和 Evaluation 结果全部重跑。
- 最终验证红灯时不冻结、不打 tag。
