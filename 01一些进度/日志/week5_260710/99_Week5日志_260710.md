## 团队完成

执行环节先补强 `tradeValidator`，关闭两类“Validator 成功、追加后 Calculator 却抛错”的接受域缺口：

- 候选交易加入完整时间线后，任一时点都不得出现负持仓。
- 未来买入不能支撑更早卖出。
- 回填卖出不能破坏后续已有卖出。
- 同时间下保持已有数组顺序，候选交易作为未来追加项排在最后。
- 候选币种必须与 Asset `quoteCurrency` 和同资产已有交易一致，否则返回 `CURRENCY_MISMATCH`。

随后新增：

- `src/services/tradeService.ts`
- `src/services/tradeService.test.ts`

公开 API 为：

```ts
createValidatedTrade(
  input: unknown,
  ledgerData: LedgerData,
  dependencies?: TradeServiceDependencies,
): CreateTradeResult
```

Service 只编排 Validator、ID 和时间依赖。校验失败返回字段错误，依赖失败返回全局 Service 错误，成功时只返回新的正式 `Trade`。ID 最多尝试 3 次；只有获得唯一 ID 后才读取一次时间。Service 不 dispatch、不调用 Calculator 或 positionService，也不修改输入和现有 `LedgerData`。
