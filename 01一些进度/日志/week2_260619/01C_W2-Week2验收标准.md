## 必须通过的计算

### 买入

输入多笔买入后，必须算出：

- 每个资产的持仓数量。
- 每个资产的成本基准。
- 每个资产的加权平均成本。

通过线：

- BTC / ETH / ADA 与手算基准一致。
- ADA 两次买入后的数量、成本、均价一致。

### 卖出

卖出按卖出前平均成本结转。

公式：

```text
soldCostBasis = sellQuantity × averageCostBeforeSell
realizedPnl = sellTotalValue - soldCostBasis
```

通过线：

- 卖出后持仓数量减少。
- 卖出后成本基准减少。
- 已实现盈亏正确。
- 剩余持仓继续保留加权平均成本口径。

### 未实现盈亏

有最新价格时：

```text
marketValue = quantity × latestPrice
unrealizedPnl = marketValue - costBasis
```

通过线：

- 没有价格快照时返回空状态，不乱算。
- 有多条价格快照时使用最新一条。
- 市值和未实现盈亏使用 Decimal 计算。

## 必须通过的校验

`TradeValidator` 必须拦截：

- 非 `buy` / `sell` 类型。
- 不存在的资产。
- `quantity <= 0`。
- `price <= 0`。
- `totalValue <= 0`。
- `fee < 0`。
- 卖出超过当前持仓。
- `quantity * price` 与 `totalValue` 明显冲突。

通过线：

- 5 条固定样例全部合法。
- 超卖 BTC 超过 `0.00016388` 必须失败。
- 校验第 5 条交易时，卖出 ADA 超过卖出前持仓 `168.2625` 必须失败。
- 5 条交易完成后，继续卖出 ADA 超过剩余持仓 `85.3244` 必须失败。
- 校验失败不能污染后续计算。

## Decimal 验收

必须满足：

- 数量、价格、总金额、手续费保存为 `DecimalString`。
- 计算统一使用 `decimal.js`。
- Decimal 操作集中在 `utils/decimalMath.ts`。
- 核心业务代码不裸写 `quantity * price`。
- 展示格式化不改变原始保存值。

通过线：

- `0.1 + 0.2` 这类浮点误差不能进入核心账本计算。
- 测试能证明 DecimalMath 的加、减、乘、除、比较可用。
