# Day 1 存算展示三段式标准

日期：2026-06-19

状态：Day1 决策记录

目标：把账本数字的“怎么存、怎么算、怎么展示”固定下来，后续所有模型、计算、校验、页面都按这个口径写。

---

## 结论

第一版统一采用三段式：

```text
保存：DecimalString
计算：decimal.js Decimal
展示：格式化字符串
```

一句话：保存值必须可还原，计算过程必须可验证，展示结果可以按场景舍入，但不能反向污染保存值。

---

## 历史依据

- Week1 数据结构已定：金额、数量、价格、手续费保存为 `DecimalString`。
- Week2 总计划已定：保存用 `DecimalString`，计算用 decimal，展示再格式化。
- `01B_W2-Decimal库选型说明` 已定：第一版选 `decimal.js`，核心计算禁止用 JS `number` 裸算。

---

## 工具结论

继续使用 `decimal.js`，暂不切换其他库。

| 候选 | 结论 | 原因 |
| --- | --- | --- |
| JS `number` | 不用 | 二进制浮点会产生金融小数误差 |
| `BigInt` / atomic integer | 不作为第一版主结构 | 适合部分资产数量，不适合价格、均价、汇率和盈亏比例 |
| `big.js` | 不选 | 更轻，但能力更窄，长期扩展余量不如 `decimal.js` |
| `bignumber.js` | 不选 | 也能做，但相对当前路线没有明显收益 |
| `decimal.js` | 选用 | 能从字符串创建，支持比较、舍入、格式化和 TypeScript 类型，适合账本核心计算 |

---

## 三段边界

| 阶段 | 数据形态 | 负责什么 | 禁止什么 |
| --- | --- | --- | --- |
| 保存 | `DecimalString` | 原始交易、价格、手续费、导入导出、加密前明文 | 存 `Decimal` 对象、存展示文案、存 JS `number` |
| 计算 | `Decimal` | 加减乘除、比较、误差判断、均价、盈亏 | 裸写 `quantity * price`，把计算细节散落到页面 |
| 展示 | formatted string | 表格、卡片、输入回显、金额符号、补零、舍入 | 用展示值继续计算，展示后再写回保存层 |

---

## 保存规则

保存层只存普通十进制字符串：

```ts
type DecimalString = string;
```

示例：

```ts
quantity: "0.00016388"
price: "67121.7"
totalValue: "11"
fee: "0"
```

规则：

- 不保存货币符号、千分位、单位、约等号、颜色、正负号文案。
- 不为了展示裁剪保存值。
- `Trade`、`PriceSnapshot`、`FeeRule` 里的精确数字字段都用 `DecimalString`。
- `Position` 第一版是计算结果，不作为原始事实保存。
- JSON 导出和 IndexedDB 保存都保持同一套 `DecimalString` 结构。

---

## 计算规则

计算只通过 `utils/decimalMath.ts` 进入 `decimal.js`：

```text
calculators / validators
  -> utils/decimalMath.ts
    -> decimal.js
```

规则：

- `calculators` 可以使用 Decimal 做中间计算。
- `validators` 可以使用 Decimal 做大小比较和误差判断。
- 对外输出计算结果时，优先回到 `DecimalString` 或明确的结果模型。
- `totalValue` 是第一版成本基准；`quantity * price` 只用于误差校验。
- 手续费第一版只记录，不计入成本和盈亏。
- `number` 只允许用于非账本金额场景，例如数组长度、UI 像素、分页数量。

---

## 展示规则

展示层拿到的是字符串结果，不直接接触 Decimal 对象。

第一版展示口径：

- 数量：保留有效小数，不强行补无意义的 0。
- USD 金额、市值、盈亏：默认展示 2 位小数。
- 均价：按资产和测试需要展示更多位，不能简单裁成 2 位。
- 测试断言：优先对比未格式化的 `DecimalString`，展示格式只测必要场景。
- 页面上的格式化值只用于看，不用于保存和二次计算。

## 自检表

| 问题 | 标准答案 |
| --- | --- |
| 数据库里保存什么 | `DecimalString` |
| JSON 导出保存什么 | `DecimalString` |
| 核心计算用什么 | `decimal.js Decimal` |
| Decimal 操作集中在哪里 | `utils/decimalMath.ts` |
| 页面能不能 `new Decimal()` | 不能 |
| 页面展示值能不能写回保存层 | 不能 |

| `quantity * price` 是成本基准吗 | 不是，只做误差校验 |
| 第一版成本基准是什么 | `totalValue` |
