## 今日完成

### S-01 保存文案与真实持久化状态对齐

- Hook 引入 `generation / mutationVersion / persistedVersion`，异步保存结果必须同时匹配 Repository generation 和 version。
- `applyLedgerAction(...)` 显式返回 `applied / noop / rejected`；只有真实 state 变化才增加 version。
- 交易与价格表单改为“已加入账本”；页面单独显示“正在保存到本地 / 已保存到本地”。
- 覆盖 applied/noop/rejected、hydration、空库、clear、旧 Promise、A 已落盘但 B pending 和旧 Repository generation。
- commit：`b609e8f 功能：区分账本变更与本地保存状态`。

### S-02 save 失败后可安全重试

- Hook 新增 `retryPersistence()`，只重试点击时的最新账本、generation 和 mutation version。
- 自动保存与手动重试复用同一写队列；重复点击共用同一 Promise。
- 新 mutation、clear、Repository 切换和组件卸载会使旧重试失效，旧 snapshot 不能排在新账本之后覆盖它。
- 页面在最新保存失败时显示“重试保存”；clear 失败不会误显示该按钮。
- commit：`97e8b23 功能：支持失败账本安全重试保存`。

### S-03 dirty 账本离开与 Repository 切换边界

- `persistedVersion !== mutationVersion` 即 dirty；pending 和 save error 都注册 `beforeunload` 警告。
- `beforeunload` 只警告，不尝试在页面关闭阶段完成异步 IndexedDB 写入。
- 最新 save/retry 成功、clear 成功或用户明确放弃后才解除 dirty。
- dirty 时收到新 Repository 会继续保留旧页面 state 并阻止切换；保存成功后自动切换，或用户点击“放弃未保存更改并切换”。
- 测试覆盖 pending、error、retry、旧 save 成功、新 mutation、clear、Repository 切换、显式放弃和卸载清理。
- commit：`401baff 功能：保护未落盘账本离开边界`。
