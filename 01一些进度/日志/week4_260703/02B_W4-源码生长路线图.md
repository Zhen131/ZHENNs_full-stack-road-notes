## 先结论

这不是日志，也不是周报，只记录源码生长关系：

```text
先长什么 -> 为什么需要 -> 它让后面什么成为可能
```

主线：

```text
DashboardShell 页面壳
-> src 分层骨架
-> models/types.ts
-> utils/decimalMath.ts
-> calculators/positionCalculator.ts
-> validators/tradeValidator.ts
-> test/fixtures.ts
-> state/initialLedgerData.ts
-> state/ledgerReducer.ts
-> 下一步 services + 页面真实接入
```

一句话：

```text
先定义账本名词，再统一小数计算，再算持仓，再拦非法交易，再建立内存账本状态。
```

## 最重要的边界

```text
Trade 是原始事实，要保存。
PriceSnapshot 是价格事实，要保存。
Position 是派生结果，不保存。
LedgerData 是当前内存账本总结构。
```

`positionCalculator.ts` 只算 `Position[]`，不读页面、不保存、不校验。

`tradeValidator.ts` 只校验输入，不生成正式 `Trade.id`，不计算仓位，不改 React state。

`ledgerReducer.ts` 只做状态变化，不校验、不计算、不读写 IndexedDB / localStorage、不保存 `Position[]`。

## 被剪掉的早期枝条

早期出现过 `ledger.ts`、`tradeDraft.ts`、`ledgerRepository.ts`、`storageAdapter.ts`、`ledgerService.ts`、`constants.ts`。

后来删掉，是因为计算和校验还没稳时，提前写 repository / service / adapter 会让项目虚胖。

现在的顺序更清楚：

```text
先算对 -> 再校验住 -> 再管理内存状态 -> 再接页面 -> 再持久化
```

## 下一步会长什么

Gate 2：`positionService.ts`，把 `LedgerData` 交给 `calculatePositions(...)` 得到 `Position[]`。

Gate 3：`DashboardShell.tsx` 停止使用写死 `trades`，改读 `LedgerData.trades`。

Gate 4：`tradeService.ts`，走 `TradeDraft -> validateTradeDraft -> Trade -> dispatch trade/add`。

Week 5：`repositories / adapters / IndexedDB / EncryptionService / JSON import-export`。

当前没有 Redux，也没有 MobX。Week 4 走 React `useReducer + LedgerData`。
