# Day 1 记录：统一 React 状态地基、总结构图与后续执行路线

日期：2026-07-05

今天是 Week4 Day1，本日没有进入源码实现，而是先把 Week4 的开发方向、学习前置和总路线图收束清楚。核心结论是：页面状态不再继续用零散 `useState` 方案，而是直接采用 `useReducer + LedgerData`，先建立 React 内存账本状态地基，再接 `tradeService`、`positionService` 和后续 IndexedDB。

---

## 今天做了什么

### 1. 学清 React 在本项目里负责什么

今天先把 React 的角色重新放回项目语境里理解：

- React 不是数据库。
- React 不是账本计算器。
- React 不是交易校验器。
- React 在 Week4 里主要负责页面状态、用户输入、dispatch 动作和展示结果。

这一步的关键理解是：页面不能继续靠写死的 `summaryRows` 和 `trades` 装样子。页面应该读一份真实的内存账本状态 `LedgerData`，然后把交易列表和资产汇总展示出来。

### 2. 确认 Week4 直接上 reducer

今天最终确认采用 `useReducer + LedgerData` 作为 Week4 状态地基。

决策原因：

- 账本后续会有很多动作：`trade/add`、`trade/delete`、`ledger/reset`、`priceSnapshot/add`、`ledger/load`、`ledger/import`。
- 如果继续使用很多零散 `useState`，后面很难判断一次用户操作到底改了账本哪里。
- reducer 能让状态变化统一变成：

```text
用户动作
-> dispatch(action)
-> ledgerReducer(state, action)
-> 新的 LedgerData
-> 页面重新展示
```

本日明确 reducer 的边界：

- reducer 只负责状态变化。
- reducer 不校验 `TradeDraft`。
- reducer 不计算 `Position[]`。
- reducer 不直接读写 IndexedDB / localStorage。
- `Position[]` 不进入 reducer state。

### 4. 重画并校准总结构图

今天重点修改了总结构图。新版结构图不再只是旧图的副本，而是把 Week4 以后真正要走的状态链路画清楚：

```text
外部输入
-> TradeDraft / PriceSnapshotDraft
-> normalize / validator / service
-> UI result handling
-> ledger actions
-> ledgerReducer
-> LedgerData
-> positionService / Repository / 后续持久化
```

新版总图补充和修正了这些关键点：

- 新增 `TradeInputSource / parser`，明确 manual form、future NLP、CSV / exchange import、future Agent draft 都只是输入来源。
- 新增 `PriceProvider / price source`，明确手动价格和未来 API 价格都不能直接写入 `LedgerData`。
- 新增 `initialLedgerData`，说明它只是页面运行态初始账本，不替代 Repository / IndexedDB。
- 新增 `ledger actions`，把账本状态变更命令独立出来。
- 将旧的 `React state / ledgerReducer` 改成更准确的 `Runtime state / ledgerReducer`。
- 新增 `UI result handling`，明确 service 成功才 dispatch，失败只显示 errors，不污染账本。
- 删除或收束旧图里重复的“安全读取路径 / 读取账本”表达。
- 给图补上 8 类颜色图例，降低后续阅读成本。

这张图现在定位为根目录级总路线图，不是 Week4 的局部附属图。

## 当前通过线

今天完成后，后续开发必须守住这几条线：

- 页面事实数据只有 `LedgerData`。
- `Position[]` 是派生结果，不保存进 state。
- 新增交易必须先变成 `TradeDraft`，校验通过后才能生成正式 `Trade`。
- Service 不直接改 reducer。
- UI 收到成功结果后 dispatch action。
- reducer 不碰校验、计算和存储。
- IndexedDB、加密、导入导出仍然是后续阶段，不进入 Week4 Day1 的源码任务。
