## Day 1：8月21日，冻结 benchmark 协议

今天只做一件事：定义测什么、怎样预置、何时开始和停止计时。

### 基础量级

- 100。
- 1,000。
- 10,000。
- 100,000 仅作为 stretch，不进入本轮硬通过线。

### 四类指标

1. 写入延迟：已有 N 条交易的账本追加一条合法 Trade，并完成加密 whole-blob 保存。
2. 查询延迟：从已解密/已 hydrate 的 N 条交易中执行锁定的查询动作；查询内容必须固定。
3. 全量 P&L 重算：只计 `calculatePositions(...)` 对 N 条交易和锁定价格快照的纯计算时间。
4. 冷启动：从提交正确口令开始，到 KDF、解密、hydrate 和核心页面 ready 结束。

### 数据预置

- 生成 N 条数据与测量写入是两件事。
- 查询、P&L 和冷启动数据集使用明确的批量测试预置路径。
- 批量预置时间不混进查询/P&L/cold-start 指标。
- 写入指标只测“已有 N 条后的下一次真实保存”，不通过 N 次 whole-blob 写构造数据集。

### 运行协议

- 固定 seed。
- 固定资产、交易分布、价格快照和 schema。
- 固定浏览器、Node、机器、构建模式和加密参数。
- 每种配置先 warm-up，再正式计时。
- Week 11 pilot 每配置 3 次。
- Week 13 final 每配置 10 次。
- 原始结果全部保留，不只保存中位数。
- 统计报告使用 median + IQR。

产出：

- 冻结版 benchmark protocol。

完成标准：

- 第三方能按协议复现实验。
- 每个指标的计时起点和终点唯一。
- pilot 开始后不为得到更好结果修改协议。
- 若修正协议缺陷，版本号变化并重跑全部 pilot。

## Day 2：8月22日，实现固定种子数据生成器

今天只做一件事：生成可复现、业务合法的交易和价格数据。

要做：

- 接收 seed、交易数和资产集合。
- 只生成生产模型允许的 buy / sell。
- 保证 sell 不超卖。
- 生成确定的时间顺序、手续费和 DecimalString。
- 生成与资产/币种匹配的价格快照。
- 相同 seed 和参数输出逐字段一致。
- 不同 seed 产生不同但合法的数据。
- 新建 100 / 1k / 10k 结构和合法性测试。

产出：

- deterministic synthetic-trade generator。

完成标准：

- 相同输入可逐字节复现。
- 全部数据通过锁定的完整账本校验。
- 不使用 Math.random 产生不可复现分支。
- 生成器本身不写 IndexedDB。

## Day 3：8月23日，建立 Playwright benchmark harness

今天只做一件事：用一条命令启动、预置、运行并保存实验。

要做：

- 使用生产 build，不测 Next.js dev 模式。
- 为每个配置创建干净浏览器环境。
- 按协议执行批量预置。
- 提供 warm-up 和正式 run。
- 为每次 run 记录 seed、scale、metric、duration、状态、环境和 commit/candidate 标识。
- 失败 run 明确记录，不静默丢弃。
- 输出原始 JSON/CSV。

产出：

- Playwright benchmark harness。

完成标准：

- 一条命令可运行最小 100 条单指标实验。
- 重复运行输出结构一致。
- 失败能定位到具体 scale / metric / run。

## Day 4：8月24日，接通写入与查询指标

今天只做一件事：实现并验证写入延迟和查询延迟。

要做：

- 按 Day 1 计时边界实现写入测量。
- 查询动作固定，不在不同 scale 改查询语义。
- 写入成功后验证数据确实持久化并可解密读取。
- 查询结果验证正确，不只记录时间。
- 每次 run 清理或恢复到一致起点。
- 新建 harness 自检和小规模稳定性测试。

产出：

- write / query benchmark。

完成标准：

- 时间不包含数据生成。
- 写入测的是实际加密 whole-blob 保存。
- 查询有正确性断言。
- 同一配置无明显初始化污染。

## Day 5：8月25日，接通 P&L 与冷启动指标

今天只做一件事：实现并验证全量计算和冷启动计时。

要做：

- P&L 指标直接调用既有 Calculator 路径，不重写计算。

- 记录加密参数和数据规模。
- 新建最小 100 条稳定性测试。

产出：

- full P&L / cold-start benchmark。

完成标准：

- 四类指标互不混淆。
- P&L 结果正确。
- 冷启动起止事件明确且可重复。
