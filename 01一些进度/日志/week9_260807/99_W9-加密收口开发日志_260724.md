## 今日完成

### 1. 关闭 P1-01 首次设密恢复死路

- 新增稳定错误码 `LEDGER_SETUP_RECOVERY_REQUIRED`。
- 首次设密写入成功但验证回读失败时，重新核对存储状态：
  - 合法 V2：保留 record，要求重新解锁。
  - 空库：返回 setup failed。
  - 读取失败：返回 read failed。
  - 非法或旧格式：返回既有结构化错误。
- 已有合法 V2 时再次提交 setup，不写入、不覆盖，直接转入恢复路径。
- Access Gate 自动进入解锁页，清空密码输入，显示“密文已写入、请重新解锁”的提示；Dashboard 不提前挂载。
- 新增 Controller 故障注入与 Gate UI 测试。
