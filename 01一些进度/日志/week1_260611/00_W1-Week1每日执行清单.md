## Day 2：6月13日，设计核心数据结构

今天只做一件事：定下数据长什么样。

需要定义：

- `Trade`：日期、类型、资产、数量、价格、手续费、备注
- `Asset`：资产代码、资产名称、计价货币
- `PriceSnapshot`：资产、价格、记录时间、来源
- `Position`：持仓数量、平均成本、已实现盈亏、未实现盈亏
- `schemaVersion`：数据版本号

产出：

- 数据结构草稿
- 交易记录清单
- 价格记录清单

完成标准：

- 看着数据结构，能判断一笔交易是否记录完整。
- 未来接实时价格 API 时，不需要推翻 `PriceSnapshot`。

## Day 5：6月16日，设计保存层和加密路线

今天只做一件事：防止页面直接碰本地存储。

需要设计：

- `LedgerRepository`：保存、读取、清空、导出、导入
- `StorageAdapter`：隔离 IndexedDB 细节
- `EncryptionService`：加密、解密、密码派生、错误处理

数据流：

```text
页面表单 -> service 校验 -> repository 保存 -> storage adapter -> 加密数据
```

产出：

- 保存层设计说明
- 加密路线说明
- 数据流说明

完成标准：

- 能指出数据从页面到本地保存会经过哪些层。
- 以后换存储方式或补加密，不需要重写页面。
