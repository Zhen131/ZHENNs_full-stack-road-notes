# Day 3 首页草图和用户流程

日期：2026-06-14

状态：第一版页面草图

目标：把 Local-First Trading Ledger 第一版页面怎么用讲清楚。今天不追求炫，也不做复杂功能，先确认一个安静、清楚、顺手的工具界面。

---

## 一句话结论

第一版首页应该是一个面向普通投资者的个人交易账本工作台。

它参考 Frappe Books 的清楚结构：

- 左侧是稳定导航。
- 右侧是当前页面内容。
- 顶部是页面标题和少量筛选。
- 主区域用清楚的分区展示核心信息。

但它不照搬会计软件的业务内容。Frappe Books 的 `Sales`、`Purchases`、`Cashflow`、`Profit and Loss`、`Top Expenses` 等模块，应该替换成投资账本自己的语言：

- 买入
- 卖出
- 交易记录
- 资产汇总
- 手动价格
- 未来的资产走势和 K 线入口

第一版的重点不是金融分析，而是让用户可以稳定完成一件事：

> 打开页面，记录一笔买入或卖出交易，再看到交易列表和资产汇总。

---

## 设计原则

### 1. 安静

页面不要一打开就给用户大量图表、指标、按钮和复杂金融术语。

第一版应该像一个干净的工作台：

- 左边告诉用户现在有哪些区域。
- 右边告诉用户当前应该看什么、做什么。
- 不用夸张的颜色。
- 不用太多动画。
- 不用一堆专业会计词。

### 2. 清楚

用户应该一眼知道：

- 我的资产现在在哪里看。
- 我在哪里新增交易。
- 我刚刚保存的交易在哪里看。
- 我在哪里手动输入当前价格。

第一版页面只保留四个核心区域：

- 资产汇总
- 新增交易
- 交易列表
- 价格输入

### 3. 顺手

顺手的意思不是第一版就完美，而是最常用动作要少绕路。

当前定义：

- 用户打开页面后，可以马上看到资产汇总。
- 用户可以明显看到“新增交易”入口。
- 新增交易时，日期默认是今天，例如 2026-06-14，但可以修改。
- 新增交易时，买入和卖出是明确的两个选项。
- 保存后，交易马上出现在交易列表。
- 手动输入价格后，资产汇总可以根据价格更新。

后续对“顺手”的定义可以继续调整。第一版先把主流程走通。

## 参考 Frappe Books 的地方

今天实际查看了 Frappe Books 打包后的软件界面。它的界面设计对本项目有参考价值。

### Frappe Books 的结构

```text
左侧导航：
Dashboard
Sales
Purchases
Common
Reports
GST
Setup

左下角辅助入口：
Help
Shortcuts
Change DB
Report Issue

右侧主区域：
顶部标题：Dashboard
右上角时间范围：This Year
第一块：Cashflow 折线图
第二块：Sales Invoice / Purchase Invoice
第三块：Profit and Loss / Top Expenses
```

### 可以借鉴的地方

- 左侧导航很稳定，不会干扰主要内容。
- 主页面标题很清楚。
- 右上角时间范围选择简单直接。
- Dashboard 被分成几块，不是杂乱堆在一起。
- 页面整体很安静，功能感强，不像营销页。
- 底部的 Help、Shortcuts、Change DB 等入口很克制。

### 不能直接照搬的地方

Frappe Books 是会计软件，服务的是公司记账、发票、采购、销售和报表。

本项目是个人投资账本，服务的是普通投资者。第一版不应该出现太多会计概念。

需要替换：

| Frappe Books | Trading Ledger |
| --- | --- |
| Dashboard | 总览 |
| Sales | 卖出 |
| Purchases | 买入 |
| Common | 交易记录 / 常用操作 |
| Reports | 报告 |
| GST | 不需要 |
| Cashflow | 资产走势占位 |
| Profit and Loss | 盈亏汇总 |
| Top Expenses | 持仓分布 / 资产占比 |
| Change DB | 切换本地账本 |

这块参考 Frappe Books 的 Cashflow 大图区域。

但第一版不做 K 线。

- 作为未来资产曲线和 K 线的位置预留。
- 告诉自己未来这里会显示“我的资产如何变化”。
- 先不要为了这块引入实时行情 API。

- 当前价格
- 计价货币，默认 USD
- 记录时间，默认现在

## 正常用户流程

### 流程 A：新增一笔买入交易

```text
1. 用户打开 Local-First Trading Ledger。
2. 默认进入总览页。
3. 用户看到资产汇总、手动价格输入、新增交易和最近交易。
4. 用户点击或定位到“新增交易”区域。
5. 交易类型默认是“买入”。
6. 用户输入资产代码，例如 BTC。
7. 用户输入数量，例如 0.00016388。
8. 用户输入成交均价，例如 67121.7。
9. 系统计算或用户填写总金额，例如 11 USD。
10. 日期默认是今天，例如 2026-06-14。
11. 如果实际交易日期不是今天，用户可以修改日期。
12. 手续费默认是 0，用户可以修改。
13. 用户可以填写备注，也可以不填。
14. 用户点击“保存交易”。
15. 系统校验字段。
16. 校验通过后，交易保存到本地。
17. 交易出现在最近交易列表。
18. 资产汇总重新计算。
```

### 流程 B：新增一笔卖出交易

```text
1. 用户打开总览页。
2. 用户进入“新增交易”区域。
3. 用户选择“卖出”。
4. 用户输入资产代码，例如 ADA。
5. 用户输入卖出数量。
6. 用户输入成交均价。
7. 日期默认今天，可以修改。
8. 用户点击“保存交易”。
9. 系统检查当前 ADA 持仓是否足够。
10. 如果持仓足够，保存交易。
11. 最近交易列表新增一条卖出记录。
12. 资产汇总重新计算当前持仓和已实现盈亏。
```

### 流程 C：手动输入当前价格

```text
1. 用户打开总览页。
2. 用户找到“手动价格输入”区域。
3. 用户选择或输入资产代码，例如 BTC。
4. 用户输入当前价格，例如 70000。
5. 计价货币默认 USD。
6. 记录时间默认现在。
7. 用户点击“保存价格”。
8. 系统保存一条 PriceSnapshot。
9. 资产汇总使用最新价格更新当前市值和未实现盈亏。
```

### 流程 D：查看结果

```text
1. 用户保存交易后，查看最近交易列表确认原始记录正确。
2. 用户保存价格后，查看资产汇总确认当前价格已更新。
3. 用户查看持仓数量、平均成本、当前市值和未实现盈亏。
4. 如果发现交易输入错了，未来可以进入交易记录修改；第一版可以先只保留修改入口占位。
```

## 错误流程

### 错误 1：必填为空

触发情况：

- 没有选择买入 / 卖出。
- 资产为空。
- 数量为空。
- 成交均价为空。
- 日期为空。

页面反馈：

```text
请填写资产、数量、成交均价和日期。
```

处理方式：

- 不保存交易。
- 把错误提示显示在表单附近。
- 保留用户已经输入的内容，避免用户重填。

### 错误 2：数字格式错误

触发情况：

- 数量不是合法数字。
- 成交均价不是合法数字。
- 总金额不是合法数字。
- 手续费不是合法数字。

错误示例：

```text
abc
1,2,3
--
0..1
```

页面反馈：

```text
数量、成交均价、总金额和手续费必须是合法数字。
```

处理方式：

- 不保存交易。
- 标出具体错误字段。

### 错误 3：数量或价格小于等于 0

触发情况：

- 数量为 0。
- 数量为负数。
- 成交均价为 0。
- 成交均价为负数。

页面反馈：

```text
数量和成交均价必须大于 0。
```

处理方式：

- 不保存交易。

### 错误 4：手续费小于 0

触发情况：

- 手续费输入负数。

页面反馈：

```text
手续费不能小于 0。
```

处理方式：

- 不保存交易。

### 错误 5：卖出超过持仓

触发情况：

用户当前只有：

```text
ADA 100
```

但尝试卖出：

```text
ADA 200
```

页面反馈：

```text
卖出数量超过当前持仓。当前 ADA 持仓为 100。
```

处理方式：

- 不保存交易。
- 提示当前可卖数量。

### 错误 6：价格输入为空或非法

触发情况：

- 资产为空。
- 当前价格为空。
- 当前价格不是合法数字。
- 当前价格小于等于 0。

页面反馈：

```text
请输入资产和大于 0 的当前价格。
```

处理方式：

- 不保存价格记录。

## 空状态设计

第一版刚打开时，可能没有任何交易。

页面不要显得坏掉，而是要告诉用户下一步做什么。

### 没有交易时

资产汇总显示：

```text
还没有资产。
添加第一笔买入交易后，这里会显示持仓、均价和盈亏。
```

最近交易显示：

```text
还没有交易记录。
请从“新增交易”开始记录你的第一笔买入或卖出。
```

### 没有价格时

资产汇总里的当前价格和未实现盈亏可以显示：

```text
未输入价格
```

手动价格输入区提示：

```text
输入当前价格后，系统会估算当前市值和未实现盈亏。
```

## Day 4 空壳实现映射

Day 4 创建网页项目时，可以先按这个页面结构搭空壳。

### 空壳数据边界

Day 4 页面里出现的资产、价格、交易记录都只是视觉占位数据，用来确认页面区域和信息结构。它们不是正式数据源，也不代表计算结果已经接入。

Week 3 接入真实状态时，必须删除 `DashboardShell` 里的硬编码数组，把交易列表、资产汇总和价格显示改成从 Week2 计算结果与 LedgerService 输出的数据渲染。

### 需要出现的文字

首页至少显示：

```text
Local-First Trading Ledger
```

### 需要出现的四个核心区域

```text
资产汇总
新增交易
交易列表
价格输入
```

### 可以额外放的未来占位

```text
资产走势
未来这里显示资产净值曲线和 K 线。
```

### 建议目录结构

```text
src/
  app/
  components/
  models/
  services/
  repositories/
  utils/
```

### Day 4 不需要实现

- 不需要真的保存交易。
- 不需要真的计算盈亏。
- 不需要接 K 线。
- 不需要接 API。
- 不需要做漂亮图表。

Day 4 只要能打开页面，并看到这些区域，就算完成。

- 新增买入和卖出都从“新增交易”表单进入。
- 日期默认今天，例如 2026-06-14，但可以修改。
- 当前价格由用户手动输入。
- 保存交易后，交易出现在最近交易列表，资产汇总重新计算。
- 必填为空、数字格式错误、数量或价格小于等于 0、手续费小于 0、卖出超过持仓都需要报错。
- K 线图、实时价格、NLP 输入、Agent 问答和复杂报告都是未来入口。
