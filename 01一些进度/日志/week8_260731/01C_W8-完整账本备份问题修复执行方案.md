### 3.1 固定复现


```text

```

### 3.2 三种 rollback 基准

分别建立测试：

| 前置状态 | import 失败后的 record |
| --- | --- |
| 无前置 save | 用户确认 import 前的 record |
| queued save 成功 | queued save 成功后的 record |
| queued save 失败 | queued save 之前最后成功的 record |

三种情况均断言页面账本、版本、dirty 和 retry 语义。

### 3.3 互斥补测

- retry 正在执行时 import 返回 `LEDGER_IMPORT_NOT_ALLOWED`，不创建第二个 operation。
- import 期间 `retryPersistence()` 返回 false。
- import 期间 clear、action 被拒绝。
- 重复 import 共享 Promise 或确定性 busy；只写一次。

## 4. 批次 B：修复 P0 持久化生命周期

修改：

```text
src/hooks/usePersistentLedger.ts
src/hooks/usePersistentLedger.test.tsx
```

### 4.1 固定实现策略

1. import 开始前不得销毁可恢复的失败状态。
2. `failedSnapshotRef` 只在 import 成功后清除。
3. active retry 存在时禁止开始 import。
4. 自动保存 effect 遇到当前 generation/version 的 `failedSnapshotRef` 时直接返回；只能由显式 retry 或新 mutation 重新调度。
5. import 失败保留 import 前：
   - failed snapshot。
   - retry 能力。
   - persistence status。
   - mutation/persisted version。
   - last persisted snapshot。
6. import 成功后统一清理失败状态，建立 clean saved baseline。

建议自动保存门禁：

```text
failedSnapshot.generation === current generation
&& failedSnapshot.version === mutationVersion
-> 不自动 enqueue
```

不得通过延时、额外 state toggle、吞掉错误或放宽测试规避第 3 次 save。

### 4.2 并发要求

- queued save 在 import 前按原顺序完成。
- queued save 成功：import 失败后以 queued save 结果为 record。
- queued save 失败：import 失败后保留最后成功 record 和 retry 状态。
- import 成功：不得触发第二次自动保存候选账本。
- Repository 切换仍延迟到 import operation 结束。
- unmount、stale token、旧 Repository completion 不修改新页面。

批次 B 通过线：

```text

原 usePersistentLedger 测试全绿
无第 3 次 save
```
