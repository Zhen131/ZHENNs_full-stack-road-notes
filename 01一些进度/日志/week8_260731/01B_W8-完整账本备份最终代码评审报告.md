## 4. 未通过项

### P0-01：导入失败会触发第 3 次非预期自动保存

位置：

```text
src/hooks/usePersistentLedger.ts:775
src/hooks/usePersistentLedger.ts:824
src/hooks/usePersistentLedger.ts:837
```

触发链：

```text
普通自动保存失败
-> failedSnapshotRef 已建立，可手动 retry
-> 开始 import 时清空 failedSnapshotRef / retryAttemptRef
-> import save 失败
-> persistenceOperation 回到 idle
-> 自动保存 effect 再次调度当前页面账本
```

独立探针实际得到 3 次 `repository.save(...)`，不是预期的“普通失败一次 + import 失败一次”。这会改变失败后的 persistence lifecycle，并可能改写 `01A` 固定的 rollback record 基准。

修复通过线：

- import 失败后恢复 import 前的失败快照、retry 能力、版本和 persistence status。
- import 失败完成不得自动调度第 3 次保存。
- 使用真实 `DefaultLedgerRepository + 故障 StorageAdapter` 断言：
  - 页面账本不变。
  - `mutationVersion / persistedVersion / isDirty / canRetryPersistence` 不变。
  - 实际 record 等于 import 前最后成功版本。
  - save 调用次数固定。

### P1-01：取消或成功后不能再次选择同一个文件

位置：

```text
src/components/backup/BackupControls.tsx:112
src/components/backup/BackupControls.tsx:246
```

取消只清理候选和组件状态，没有清空 `<input type="file">` 的 value。浏览器把第二次同文件选择视为未变化，不触发 `onChange`。

修复通过线：

- 使用 input ref；选择处理后、取消后、成功后均重置 value。
- 增加“同文件取消后重选”和“成功后同文件重选”测试。
