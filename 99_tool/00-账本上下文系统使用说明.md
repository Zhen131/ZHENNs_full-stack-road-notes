# 账本上下文系统使用说明

更新时间：2026-06-25
状态：已安装

## 你得到了什么

这套系统让新的 Codex 聊天框自动知道：

- 账本源码、日志、规划和外部参考分别在哪里。
- Frappe Books 不是你的源码。
- 根目录和账本源码是两个独立 Git 仓库。
- 当前已经做到哪里，下一步是什么。
- 不应该默认读取哪些无关或私人文件。

## 日常怎么用

打开新聊天框后，直接说一句：

```text
账本写代码：今天实现未实现盈亏
账本 Git：检查当前改动并整理提交
账本讨论：页面内存状态应该怎么组织
账本继续
账本日志：总结今天完成的代码和测试
```

也可以直接描述任务：

```text
帮我把最新价格接到 positionCalculator。
看看账本源码现在能不能提交。
我们讨论一下 IndexedDB repository 的接口。
```

不需要再次说明源码路径、日志路径、当前周、Frappe Books 的性质或两个 Git 仓库的区别。

| 文件 | 为什么不放在 `99_tool/` |

| 文件 | 作用 |

## 系统怎样工作

```text
你说今天的任务
→ Codex 自动读取 AGENTS.md
→ ledger-workflow 识别任务模式
→ 读取当前开发状态
→ 只找任务直接相关的文件
→ 执行并验证
→ 有实质进展时更新当前状态
```

## 当前状态什么时候更新

应该更新：

- 完成一个代码功能或修复。
- 测试结果发生变化。
- 当前里程碑、下一步或风险发生变化。
- 关键入口文件发生变化。

不应该更新：

- 只是讨论方案。
- 只读检查文件。
- 没有形成可验证进展。
- 只是计划以后要做什么。

## 如果 AI 找错了上下文

直接说：

```text
重新按账本工作流定位，只读取当前状态和本任务相关文件。
```

如果是 Git 任务，再补一句目标：

```text
目标是账本源码仓库。
```

或：

```text
目标是根目录文档仓库。
```

## 维护原则

- 动态进度只维护在 `00-当前开发状态.md`，不要复制到 `AGENTS.md` 或 Skill。
- 新增永久目录规则时修改 `AGENTS.md` 和 AI 指南。
- 新增任务模式或读取流程时修改 `ledger-workflow` Skill。
- Skill 修改后重新运行官方 `quick_validate.py`。
- `02_NLP/` 保持原位，等账本进入 NLP 接入阶段再启用。
