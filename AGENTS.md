# 账本项目协作规则

## 项目范围

- 本工作区默认服务于 Local-First Trading Ledger，包括开发、Git、架构讨论、日志和论文素材。
- 用户直接描述账本任务，或使用“账本写代码 / 账本 Git / 账本讨论 / 账本继续 / 账本日志”时，按 `.agents/skills/ledger-workflow/SKILL.md` 执行。
- 开始任务时先判断工作模式，再读取最小必要上下文；不要为了熟悉项目遍历整个工作区。

## 固定入口

1. 动态进度：`01一些进度/日志/00-当前开发状态.md`
2. 账本源码：`01一些进度/产出/LocalFirstTradingLedger/`
3. 当前周计划：`01一些进度/日志/week*_*/`
4. 长期原则：`00开始前的一些准备文件/00A-全栈之路AI助手使用指南.md`
5. 系统设计与使用说明：`99_tool/`

只在任务需要时继续读取相关代码、测试、设计或历史日志。

## 目录与读取边界

- `00frappe-books-typescript/` 是外部参考项目，不是用户账本源码。默认不读，只在明确比较架构或用户要求时读取相关文件。
- `02_NLP/` 是后续要接入账本的模块。非 NLP 任务默认不读，也不要移动或改写。
- `*.canvas` 只在架构图、数据流或用户明确要求时读取。
- `00开始前的一些准备文件/99-私人网络环境说明.md` 永不主动读取、引用或提交。

## Git 边界

- 工作区根目录是文档与计划仓库。
- `01一些进度/产出/LocalFirstTradingLedger/` 是独立源码仓库。
- Git 任务先确认目标仓库，再查看该仓库的 `status`、分支、日志和 diff；不得混用两个仓库的提交。
- 保留用户已有改动。除非用户明确要求，不使用破坏性 Git 命令，不擅自提交或推送。

## 执行与收尾

- 写代码时先定位相关实现与测试，修改后运行与风险相称的测试、lint 或 build。
- 讨论任务默认只读；没有明确实施要求时不修改代码或 Git 状态。
- 只在出现实质进展、里程碑变化或下一步变化时更新当前开发状态。
- 文档遵循 `99_tool/00-文档产出规范.md`：先结论，写清规则、边界和通过线。
- 回答保持简洁直接，明确结果、验证证据和下一步。
